# 문장순간 v0.6 - OCR/AI/템플릿 추천 명세

## 1. 목적

OCR, AI 분위기 분류, 템플릿 추천 엔진의 역할과 한계를 명확히 정의한다.

## 2. OCR 설계

### 처리 흐름

```text
이미지 업로드
  → 이미지 전처리
  → OCR 실행
  → 텍스트 블록 반환
  → 문장 단위 분리
  → 원본 이미지 삭제
  → 사용자가 선택한 짧은 문구만 저장 가능
```

### OCR 원칙

| 원칙 | 설명 |
|---|---|
| OCR은 초안 | 사용자가 반드시 확인/수정할 수 있어야 한다. |
| 전체 원문 미저장 | OCR 전체 텍스트는 임시 세션으로만 유지한다. |
| 1회 1장 | MVP에서는 여러 페이지 업로드를 허용하지 않는다. |
| 실패 fallback | 재촬영 또는 직접 입력 모드를 제공한다. |

## 3. AI 역할

AI는 다음만 수행한다.

- 분위기 mood 분류
- genreTone 분류
- visualKeywords 생성
- recommendedTemplateTags 생성
- 선택적으로 카드 제목 후보 생성

AI는 다음을 수행하지 않는다.

- OCR
- 전체 페이지 분석
- 책 전체 요약
- 선택 문구 재작성
- 배경 이미지 생성
- 렌더링

## 6. 추천 점수식

```text
score =
  moodMatch * 40
  + genreMatch * 15
  + textLengthFit * 20
  + ratioSupport * 15
  + emphasisMatch * 10
```

## 7. 추천 규칙

| 조건 | 처리 |
|---|---|
| 문구 121자 이상 | 추천 차단 |
| 선택 비율 미지원 | 후보 제외 |
| AI 실패 | 기본 mood = 차분함, 기본 템플릿 추천 |
| 추천 후보 0개 | 비율/강조 방식 재선택 유도 |
| 동일 점수 | 최근 덜 사용한 템플릿 우선 |

## 8. 의사코드

```ts
function getTextLength(text: string) {
  const len = [...text].length;
  if (len <= 40) return 'short';
  if (len <= 90) return 'medium';
  if (len <= 120) return 'long';
  return 'blocked';
}

function recommendTemplates(input, templates) {
  const textLength = getTextLength(input.selectedQuote);
  if (textLength === 'blocked') return [];

  return templates
    .filter(t => t.ratios.includes(input.ratio))
    .map(t => ({
      template: t,
      score: calculateScore(input, t, textLength)
    }))
    .sort((a, b) => b.score - a.score)
    .slice(0, 3);
}
```
