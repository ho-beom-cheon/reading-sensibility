# Codex Task 04: Create flow UI shell

Goal: implement the user-visible create flow shell without external integrations.

Screens:

1. Home
2. Upload
3. OCR processing
4. Quote selection/editing
5. Details input
6. Template selection
7. Preview/save
8. Done

Requirements:

- Use the copy and constraints from design docs.
- Show copyright/privacy notices on upload and save screens.
- Support direct input fallback.
- Prevent next step when required data is missing.
- Keep UI mobile-first.

Acceptance criteria:

- User can move through the flow using mock data.
- Required validation is visible.
- Notices are visible.
- No external API calls are required.
